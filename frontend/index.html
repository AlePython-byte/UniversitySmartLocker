Contexto del caso de estudio

Resumen del problema académico que modela este proyecto.

¿Qué representa este sistema?

Este sistema representa un casillero inteligente universitario para la entrega y recogida segura de paquetes. El casillero cambia su comportamiento según su estado actual, por ejemplo: disponible, reservado, con paquete guardado, listo para recoger, vencido o en mantenimiento.

¿Por qué se usa el Patrón State?

Se aplica el Patrón State porque cada estado controla qué acciones están permitidas y cuáles no. De este modo se evita concentrar toda la lógica en condicionales complejos y las transiciones quedan explícitas y fáciles de explicar.

Flujo del casillero

Secuencias principales que se pueden explicar visualmente durante la demostración del patrón.

Disponible → Reservado → Paquete guardado → Listo para recoger → Disponible
Listo para recoger → Vencido → Mantenimiento → Disponible
