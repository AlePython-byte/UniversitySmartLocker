Contexto del caso de estudio

Descripción académica del sistema y del motivo por el cual se aplica el Patrón State.

Este sistema representa un casillero inteligente universitario para la entrega y recogida segura de paquetes. El casillero cambia su comportamiento según su estado actual: disponible, reservado, con paquete guardado, listo para recoger, vencido o en mantenimiento. Por eso se aplica el Patrón State, ya que cada estado define qué acciones están permitidas y cuáles no.

Resumen del flujo del casillero

Secuencias principales que puedes probar desde la interfaz.

Disponible → Reservado → Paquete guardado → Listo para recoger → Disponible
Listo para recoger → Vencido → Mantenimiento → Disponible
